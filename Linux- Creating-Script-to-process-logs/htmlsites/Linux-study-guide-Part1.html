Linux Study Guide

1. Log Processing and Script Creation in Linux

Log files store records of events, errors, and activities in an application. Monitoring logs helps identify issues.

Creating a Log Monitoring Script

mkdir scripts-linux
cd scripts-linux

Creating a Script File using Vim

vim log-monitoring.sh

Basic Script Structure

#!/bin/bash
LOG_DIR="../myapp/logs"
echo "Checking logs in directory $LOG_DIR"

Save and Exit Vim

Press Esc, type :wq, and hit Enter.

2. Managing Permissions in Linux

Grant Execution Permission

chmod 755 log-monitoring.sh

Octal Notation Explanation

  • 7 (User): Read (4) + Write (2) + Execute (1) = rwx
  • 5 (Group): Read (4) + Execute (1) = r-x
  • 5 (Others): Read (4) + Execute (1) = r-x

Symbolic Notation Example

chmod u=rwx,g=rx,o=rx log-monitoring.sh

Check File Permissions

ls -l

Running the Script

./log-monitoring.sh

3. User and Group Management in Linux

Create a New User

sudo adduser julia

Create a Group and Add Users

sudo groupadd devs
sudo usermod -aG devs julia

Verify User Groups

groups julia

Verify Group Creation

getent group devs

4. Adjusting Directory Permissions

Change Group Ownership of a Directory

sudo chown -R :devs /home/user/scripts-linux

Check Directory Permissions

ls -ld /home/user/scripts-linux

Switch to Julia's Account and Execute the Script

su - julia
cd /home/user/scripts-linux
./log-monitoring.sh

Conclusion

Now you have a good understanding of:

  • Creating and executing Bash scripts in Linux
  • Configuring permissions correctly with chmod
  • Creating and managing users and groups
  • Adjusting file and directory permissions for collaboration
